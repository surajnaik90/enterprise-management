# Performance domain

It is about practical actions necessary to achieve project success.

## Project principles  
1. It is about ethical and behavioral guidelines on how project is planned, executed and delivered.   
2. It establishes behavioral frameworks.

## 8 gears of performance domain

1. Stakeholder engagement   
2. Team   
3. Development approach and lifecycle   
   a. Tailoring approach   
   b. Flexibility and adaptibility   
4. Planning   
5. Project work   
6. Delivery   
7. Measurement   
8. Uncertainity   

### Benefits of domains

1. Increased focus on outcomes   
2. Improved integration   
3. Enhanced alignment and objectives.  

## Stakeholder performance domain

3 Key areas:

1. **Identify and analyze stakeholders**   
   a. Create a stakholder register.   
   b. Roles. Responsibilities.   
2. **Developing engagement strategies**   
   a. Clear communications on when and how.   
   b. Needs. Expectations. Involvement. Participation.   
3. **Managing stakeholder relationships**

Stakeholder matrix:   

It's the power interest grid of stakeholders.

## Team performance domain

1. Team culture, collaboration, adaptibility, shared leadership, ownership.
2. Eliminate barriers. 
3. Mulit faceted approach.

### Tuckman model

1. **Forming**: Get to know each other. Roles. Expectations. Strucutre. Set groundwork.   
2. **Storming**: Personalities & working styles emerge.   
3. **Norming**: Time of collaboration. Comfortable. Everyone aligning on roles and responsibilities.
4. **Performing**: Team reached peak performance.   
5. **Adjourning**: Reflect. Celebrate. Closure. Accomplishments. Transition.
